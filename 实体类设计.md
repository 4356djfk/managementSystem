# 软件项目管理工具实体 / DTO / VO 设计

## 1. 设计目标

本文档基于现有 `API接口定稿.md`、`功能点定稿.md`、`数据库结构设计.md`，统一定义后端开发所需的数据对象模型，作为以下工作的基线：

1. 数据库表结构与 Entity 建模
2. Controller 请求参数 DTO 设计
3. Service 业务对象传递规范
4. 接口响应 VO 设计
5. Knife4j / OpenAPI 文档编写依据

默认约定：

1. Java 包结构建议：
   - `com.manage.managesystem.entity`
   - `com.manage.managesystem.dto`
   - `com.manage.managesystem.vo`
   - `com.manage.managesystem.enums`
2. 主键统一为 `Long id`
3. 时间字段优先使用 `LocalDateTime`，纯日期使用 `LocalDate`
4. 金额字段使用 `BigDecimal`
5. 删除策略默认逻辑删除，Entity 保留 `deleted`
6. 统一返回结构使用 `ApiResponse<T>`
7. 分页结构使用 `PageResult<T>`

## 2. 分层约定

### 2.1 Entity

Entity 对应数据库表，字段尽量与数据库列一一对应，不直接暴露给前端。

### 2.2 DTO

DTO 用于接口入参，按职责拆分：

1. `CreateXxxDto`
2. `UpdateXxxDto`
3. `QueryXxxDto`
4. `PatchXxxDto`

### 2.3 VO

VO 用于接口出参，面向前端页面展示，允许聚合名称、统计值、嵌套对象。

## 4. 枚举定义建议

### 4.1 用户与权限

1. `SystemRoleEnum`: `PROJECT_MANAGER` / `TEAM_MEMBER` / `READ_ONLY` / `SYS_ADMIN`
2. `UserStatusEnum`: `ACTIVE` / `INACTIVE` / `LOCKED`
3. `ProjectMemberStatusEnum`: `ACTIVE` / `REMOVED`

### 4.2 项目管理

1. `ProjectStatusEnum`: `PLANNING` / `IN_PROGRESS` / `MONITORING` / `CLOSED`
2. `LifeCycleModelEnum`: `WATERFALL` / `AGILE` / `ITERATIVE` / `INCREMENTAL`
3. `ProjectTemplateStatusEnum`: `ENABLED` / `DISABLED`

### 4.3 范围与进度

1. `RequirementTypeEnum`: `FUNCTIONAL` / `NON_FUNCTIONAL`
2. `RequirementStatusEnum`: `DRAFT` / `CONFIRMED` / `CHANGED` / `CLOSED`
3. `TaskTypeEnum`: `TASK` / `SUB_TASK` / `MILESTONE_TASK`
4. `TaskStatusEnum`: `TODO` / `IN_PROGRESS` / `BLOCKED` / `DONE`
5. `PriorityEnum`: `LOW` / `MEDIUM` / `HIGH` / `URGENT`
6. `MilestoneStatusEnum`: `PENDING` / `REACHED` / `DELAYED`
7. `DependencyTypeEnum`: `FS` / `SS` / `FF` / `SF`

### 4.4 变更、风险、成本、通知

1. `RiskLevelEnum`: `LOW` / `MEDIUM` / `HIGH` / `CRITICAL`
2. `RiskStatusEnum`: `IDENTIFIED` / `ANALYZED` / `RESPONDING` / `CLOSED`
3. `ChangeTypeEnum`: `SCOPE` / `SCHEDULE` / `COST` / `QUALITY` / `CONFIG` / `OTHER`
4. `ChangeStatusEnum`: `SUBMITTED` / `UNDER_REVIEW` / `APPROVED` / `REJECTED` / `IMPLEMENTED`
5. `ExportModuleEnum`: `TASK` / `RISK` / `COST` / `TIMESHEET` / `REPORT`
6. `ExportFormatEnum`: `EXCEL` / `PDF`
7. `NotificationReadStatusEnum`: `UNREAD` / `READ`

## 10. 建议落地顺序

1. 先落 `BaseEntity`、`ApiResponse`、`PageResult`
2. 先实现第一批核心模块：用户、项目、成员、WBS、任务、风险、变更、工时
3. Entity 命名与数据库表严格对齐
4. DTO 按接口动作拆分，不要复用一个大而全的请求类
5. VO 面向页面输出，不直接回传 Entity
