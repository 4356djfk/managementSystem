# 软件项目管理系统实体 / DTO / VO 设计

## 1. 分层约定

### 1.1 Entity

目录：`com.manage.managesystem.entity`

用途：映射数据库表字段，承载持久化模型。

### 1.2 DTO

目录：`com.manage.managesystem.dto`

用途：接口入参模型，按创建、更新、查询、审批、状态变更等场景拆分。

### 1.3 VO

目录：`com.manage.managesystem.vo`

用途：接口出参模型，面向前端展示与聚合结果。

## 4. 最近更新的关键模型

### 4.1 TaskEntity

当前任务实体除了常规计划与实际字段，还包含新的进度控制字段：

1. `deadlineDate`
2. `constraintType`
3. `constraintDate`

任务仍保留：

1. 父任务 `parentTaskId`
2. WBS 关联 `wbsId`
3. 里程碑关联 `milestoneId`
4. 负责人 `assigneeId`
5. 工时与成本聚合字段

### 4.2 TaskDependencyEntity

除前置、后置、依赖类型外，已新增：

1. `lagDays`

用于表示依赖关系的滞后时间。

### 4.3 ProjectEditorPreferenceEntity

项目编辑器偏好已独立建模：

1. `ganttAppearanceJson`
2. `wbsConfigJson`
3. `scheduleOptionsJson`

该模型用于保存前端项目编辑器的展示与排程配置。
