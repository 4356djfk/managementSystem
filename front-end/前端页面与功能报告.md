# 前端页面与功能报告

## 1. 文档目的

本文档基于当前后端**已实现接口**，整理前端应建设的页面、页面功能点、建议路由与接口映射，供前端开发直接拆解任务。

当前前端工程为：

- 技术栈：`Vue 3 + Vite + Vue Router + Pinia`
- 当前状态：工程已初始化，但业务页面、路由和状态管理基本未落地

## 2. 建议整体信息架构

建议前端采用以下一级导航：

1. 登录
2. 首页仪表盘
3. 项目中心
4. 项目模板
5. 用户与权限
6. 任务与进度
7. 风险与变更
8. 成本与工时
9. 收尾与报告
10. 通知与搜索
11. 系统审计

## 4. 页面清单与功能点

### 4.1 登录页

页面名称：
- 登录页

核心功能：
- 用户名密码登录
- 登录后保存 token
- 获取当前用户信息
- 根据角色控制菜单显示

对应接口：
- `POST /auth/login`
- `GET /auth/me`
- `POST /auth/logout`

建议页面元素：
- 用户名输入框
- 密码输入框
- 登录按钮
- 登录失败提示

## 8. 结论

基于当前后端接口，前端已经可以支撑一套较完整的软件项目管理系统页面建设，核心业务主线已经具备：

- 登录与权限
- 项目全生命周期管理
- 范围/进度/成本/风险/变更/工时
- 收尾、归档、报告
- 通知、搜索、审计

当前最适合前端的工作方式是：

1. 先完成基础布局、登录态、菜单权限
2. 再按“项目中心”主线逐个模块落页面
3. 最后补平台型页面，如通知、搜索、审计、导入导出
