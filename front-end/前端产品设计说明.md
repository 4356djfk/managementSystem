# 前端产品设计说明

## 1. 设计目标

前端不是单纯展示页，而是一个面向项目经理和项目成员的“项目工作台”。

设计目标：

1. 让用户先在仪表盘完成项目选择和全局操作
2. 再进入项目编辑器进行高密度业务编辑
3. 在同一工作区内整合计划、执行、监控和结项能力

## 2. 信息架构

### 2.1 双工作区结构

1. 仪表盘工作区：全局项目入口、系统管理入口、通知与个人信息
2. 项目编辑器工作区：单项目全量业务操作

### 2.2 主要用户路径

1. 登录
2. 进入 `/dashboard`
3. 查看项目概览和待办
4. 进入 `/editor/:projectId`
5. 在编辑器中维护项目全过程数据

## 3. 当前页面分布

### 3.1 仪表盘页

承载：

1. 项目总览
2. 快速入口
3. 当前用户关注任务
4. 里程碑与工时提醒

### 3.2 系统管理页

仅面向管理员：

1. 模板中心
2. 用户管理
3. 审计日志

### 3.3 业务通用页

1. 我的日历
2. 全局搜索
3. 通知中心
4. 个人资料

### 3.4 项目编辑器页

这是产品核心页面，负责整合：

1. 项目基本信息
2. 项目章程
3. 团队成员
4. WBS 与需求
5. 任务、里程碑、依赖、甘特图
6. 风险、成本、质量、测试
7. 沟通、采购、配置、变更
8. 工时、报告、归档、验收

## 4. 交互特点

### 4.1 高密度编辑

项目编辑器采用高信息密度布局，适合桌面端持续作业，而不是单次填表。

### 4.2 模块化弹窗

复杂明细通过多个对话框承载，例如：

1. 任务详情
2. 里程碑
3. 风险
4. 质量计划
5. 测试计划
6. 测试用例
7. 缺陷
8. 风险矩阵
9. 沟通矩阵
10. WBS 编辑

### 4.3 偏好持久化

项目编辑器已支持保存用户在项目维度上的展示偏好：

1. 甘特外观
2. WBS 配置
3. 排程选项

## 5. 最近设计演进

前端已从基础 CRUD 界面演进到项目排程工作台，主要变化包括：

1. 引入甘特视图与任务排程
2. 补齐质量、测试、采购、配置等辅助模块
3. 增加导入导出与附件能力
4. 增加任务计划约束、截止日期、依赖滞后等排程字段
5. 增加项目级编辑器偏好存储

## 6. 产品结论

当前前端设计应被视为“项目工作台 + 系统管理后台”的组合，而不是单页面项目展示应用。
