# 前端页面分布与界面风格说明

## 1. 页面分布原则

当前项目已经形成清晰的两层结构：

1. `/dashboard` 负责全局入口和系统级能力
2. `/editor/:projectId` 负责单项目深度编辑

这套结构适合继续沿用，不建议把所有功能重新拆回多个松散页面。

## 2. 当前界面风格判断

项目现状更接近“管理工作台 + 桌面式编辑器”，而不是营销型后台。

因此界面风格应继续坚持：

1. 信息密度高
2. 强调表格、侧栏、面板与时间轴
3. 重点突出任务、计划、风险、成本这些项目管理核心对象
4. 减少大面积空白和装饰性卡片堆叠

## 3. 项目编辑器的视觉方向

项目编辑器建议持续向如下方向收敛：

1. 顶部为功能分组与命令入口
2. 左侧 / 主区承载 WBS 与任务结构
3. 中央区域承载甘特与排程
4. 右侧或弹窗承载对象详情与编辑表单

## 4. 当前已适配的排程能力

界面需要继续体现以下字段，而不是只展示任务名和时间段：

1. 截止日期
2. 计划约束类型
3. 计划约束日期
4. 任务依赖类型
5. 依赖滞后天数
6. 关键路径标识
7. 进度预警状态

## 5. 色彩与控件建议

### 5.1 推荐方向

1. 主色保持冷静专业，适合项目管理场景
2. 状态色区分明确：正常、风险、阻塞、完成
3. 甘特视图应支持主题切换，但不要弱化边界和依赖线

### 5.2 已有能力

项目编辑器已经具备甘特外观配置持久化，因此风格扩展应优先围绕：

1. 甘特条颜色
2. 周末背景
3. 摘要任务样式
4. 里程碑样式

## 6. 页面组织建议

### 6.1 保持独立页面

应继续保持独立入口的页面：

1. 仪表盘
2. 用户管理
3. 模板中心
4. 日历
5. 搜索
6. 审计日志
7. 通知中心
8. 个人资料

### 6.2 保持集成到编辑器

更适合继续留在项目编辑器中的模块：

1. WBS
2. 任务与甘特
3. 风险
4. 成本
5. 质量
6. 测试
7. 配置
8. 变更
9. 沟通
10. 采购
11. 工时
12. 报表
13. 归档与验收

## 7. 结论

项目当前的最佳风格参考不是“普通管理后台”，而是“偏工程项目排程工具”的工作台。后续界面优化应围绕项目编辑器的可读性、可排程性和信息组织效率继续加强。
