# 前端页面分布与界面风格建议（对标 Microsoft Project）

## 1. 目标说明

当前系统后端接口已经覆盖了项目管理主流程。  
前端如果希望更接近 **Microsoft Project / Project for the web** 的使用习惯，重点不在“把所有功能平铺出来”，而在于：

1. 功能入口按项目管理流程组织
2. 页面尽量以“工作区”而不是“普通后台列表页”来设计
3. 核心模块优先采用表格、时间轴、侧栏详情这类专业项目管理界面
4. 视觉风格保持简洁、偏企业级、信息密度高

## 2. 总体导航建议

不建议采用传统“很多独立菜单页”的后台分布。  
建议改成更接近 Microsoft Project 的分层方式：

### 一级导航

1. 主页
2. 项目
3. 资源与成员
4. 报告
5. 系统

### 二级结构建议

#### 主页

- 首页仪表盘
- 我的任务
- 我的日历
- 我的通知

#### 项目

- 项目列表
- 项目模板
- 项目详情工作区

#### 资源与成员

- 用户管理
- 项目成员管理
- 工时管理

#### 报告

- 周报/月报
- 项目总结报告
- 审计日志
- 导出任务

#### 系统

- 搜索
- 附件
- 导入导出

## 3. 最核心的变化：项目详情不要做成“很多独立页面”，而要做成“项目工作区”

这一点最像 Microsoft Project。

### 建议的项目详情工作区结构

路由建议：

```text
/projects/:projectId/workspace
```

页面布局建议：

1. 顶部：项目名称、状态、负责人、时间范围、预算、操作按钮
2. 左侧：项目内部导航
3. 中间：主工作区
4. 右侧：详情抽屉 / 属性面板

### 左侧项目内部导航建议

1. 概览
2. 任务
3. 甘特图
4. 里程碑
5. WBS
6. 需求
7. 风险
8. 变更
9. 成本
10. 工时
11. 日历
12. 收尾
13. 报告

这比把每个功能散落到一级菜单更接近 Microsoft Project 的“围绕单个项目展开工作”模式。

## 4. 页面分布如何向 Microsoft Project 靠拢

## 4.1 首页

不要做成花哨大屏。  
应更像“我的工作台”。

建议分区：

1. 我的待办任务
2. 项目总体概览
3. 即将到来的里程碑
4. 风险与变更提醒
5. 最近访问项目

对应感觉：
- 像 Microsoft 365 / Project 首页那种工作入口页

## 4.2 项目列表页

建议偏“项目卡片 + 列表切换”而不是纯后台表格。

推荐布局：

1. 顶部筛选栏
2. 视图切换：卡片视图 / 表格视图
3. 项目卡片显示：
   - 项目名称
   - 状态
   - 负责人
   - 计划日期
   - 进度条
   - 风险/变更数量

更接近 Microsoft Project 的原因：
- 用户先进入项目，再进入项目工作区

## 4.3 任务页

这是最需要靠近 Microsoft Project 的核心页面。

建议默认提供三种视图切换：

1. 网格视图
2. 看板视图
3. 甘特视图

### 网格视图

适合替代传统表格。

建议列：
- 任务名称
- 状态
- 优先级
- 负责人
- 开始时间
- 结束时间
- 工时
- 进度
- 里程碑

交互建议：
- 行内编辑
- 多选批量操作
- 点击行打开右侧详情抽屉

### 看板视图

按状态分列：
- TODO
- IN_PROGRESS
- BLOCKED
- DONE

这更像 Planner / Project for the web 的体验。

### 甘特视图

左侧任务树 + 右侧时间轴

这是最接近 Microsoft Project Desktop 的核心界面。

建议：
- 支持展开父子任务
- 支持显示依赖线
- 点击任务在右侧抽屉查看详情

## 4.4 资源与成员页

这里可以参考 Microsoft Project 的“资源管理”思路。

建议不要只做“成员列表”。

项目成员页建议分成两块：

1. 成员清单
2. 成员工作负载摘要

显示信息：
- 姓名
- 角色
- 项目状态
- 已分配任务数
- 工时汇总

## 4.5 风险与变更页

这两个页面建议保留企业项目管理风格，而不是做成轻量工具风格。

### 风险页

建议布局：

1. 上方筛选
2. 左侧风险列表
3. 右侧详情
4. 下方或切换页签显示风险矩阵

### 变更页

建议布局：

1. 变更列表
2. 变更详情
3. 审批记录时间线

这类页面更接近 Microsoft 体系里“企业工作流”页面风格。

## 4.6 成本页

建议做成页签式工作区：

1. 成本计划
2. 实际成本
3. 成本基线
4. EVM 指标

其中：

- 成本计划：表格为主
- 实际成本：明细记录为主
- 成本基线：历史版本列表
- EVM：卡片 + 小图表

微软风格更强调：
- 表格密度高
- 指标清晰
- 不做过度装饰

## 4.7 工时页

建议分成：

1. 我的工时
2. 项目工时
3. 工时报表

其中“我的工时”应尽量简单，类似周历或日历录入。  
“项目工时”则偏管理视图。

## 4.8 收尾页

不要拆得太散，建议做成一个“项目收尾工作区”。

页签建议：

1. 验收项
2. 文档归档
3. 经验教训
4. 关闭前检查

这样更符合项目经理在收尾阶段连续处理事项的习惯。

## 4.9 报告页

建议从“文档列表”思路改成“报告中心”。

分区建议：

1. 报告生成区
2. 报告记录列表
3. 总结报告入口

更像 Microsoft 体系里的“输出结果中心”。

## 5. 界面样式建议：尽量向 Microsoft Project 靠拢

## 5.1 视觉基调

建议关键词：

- 简洁
- 稳重
- 高信息密度
- 偏浅色企业风
- 轻边框、低饱和配色

### 推荐风格

1. 主背景：浅灰白
2. 卡片背景：纯白
3. 强调色：蓝色系
4. 状态色：绿 / 黄 / 红 / 灰
5. 圆角：小圆角，不要太大
6. 阴影：弱阴影，避免浮夸

### 不建议

1. 大面积渐变
2. 夸张动效
3. 花哨插画首页
4. 过大的卡片和留白

## 5.2 组件风格建议

### 表格

要作为核心组件来设计。

要求：

1. 支持固定列
2. 支持列宽调整
3. 支持排序筛选
4. 支持行选中高亮
5. 支持行内编辑

### 详情抽屉

Microsoft Project / Azure DevOps 风格里很常见。  
建议大量使用右侧抽屉，而不是频繁整页跳转。

适用场景：

1. 任务详情
2. 风险详情
3. 变更详情
4. 成本记录详情

### 过滤器栏

建议统一成顶部工具条样式：

1. 关键字搜索框
2. 下拉筛选
3. 日期范围
4. 视图切换
5. 新建按钮

## 5.3 布局风格建议

推荐：

1. 顶部全局导航 + 左侧模块导航 + 主内容区
2. 项目内采用局部左侧导航
3. 大量使用“表格 + 时间轴 + 右侧属性面板”

这比传统后台“纯列表 + 弹窗”更专业。

## 6. 推荐页面原型优先级

如果要先做一版最像 Microsoft Project 的前端，建议优先顺序如下：

### 第一优先级

1. 登录页
2. 首页工作台
3. 项目列表页
4. 项目工作区
5. 任务网格页
6. 甘特图页

### 第二优先级

1. 风险页
2. 变更页
3. 成本页
4. 工时页
5. 成员页

### 第三优先级

1. 收尾页
2. 报告中心
3. 通知中心
4. 搜索页
5. 审计日志

## 7. 对前端实现的直接建议

结合当前 `Vue 3` 项目，建议前端下一步这样落：

1. 先搭一个 `AppShell`
2. 做全局顶部导航
3. 做项目工作区布局
4. 先实现“项目列表 -> 项目工作区 -> 任务网格/甘特图”
5. 之后再平移到风险、变更、成本、工时

## 8. 结论

如果前端目标是“更像 Microsoft Project”，最重要的不是把接口一页页机械映射，而是：

1. 让“项目”成为核心工作上下文
2. 让“任务 + 甘特图”成为最核心主界面
3. 让风险、变更、成本、工时围绕项目工作区组织
4. 用企业级、低装饰、高密度的视觉风格来承载复杂信息

相比普通后台管理系统，更推荐你们把这个产品做成：

- 首页工作台
- 项目工作区
- 专业任务视图
- 项目管理专题页

而不是“很多分散的 CRUD 页面”。
