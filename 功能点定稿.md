# 软件项目管理工具功能点定稿

## 1. 文档目标

本文档基于《软件项目管理工具.docx》整理，输出一版可直接用于产品设计、数据库设计、接口设计和开发排期的功能点清单。

本文档按 `V1.0 必做`、`V1.1 推荐`、`V1.2 可选` 三个层级冻结范围，默认以 `V1.0 + V1.1` 作为本项目正式交付范围。

## 2. 产品定位

本系统是一个面向课程实验和中小型软件项目的项目管理平台，覆盖项目从立项、计划、执行控制到结项的完整流程，并提供角色权限、日志审计、报表导出和仪表盘等基础能力。

## 3. 用户角色

### 3.1 角色定义

| 角色 | 典型用户 | 权限说明 |
| --- | --- | --- |
| 系统管理员 | 平台管理员、运维负责人 | 管理用户、系统角色、模板、审计和系统级配置，不直接参与项目业务 |
| 项目经理 | 课程负责人、项目负责人 | 创建项目、维护计划、审批变更、查看所有项目数据、归档和关闭项目 |
| 团队成员 | 开发、测试、产品、实施 | 查看参与项目、更新自己任务进度、登记工时、提交变更、登记风险、参与协作 |
| 只读访客 | 老师、助教、客户 | 查看项目计划、进度、报表和仪表盘，不允许修改 |

### 3.2 权限原则

1. 项目经理对所属项目拥有全量管理权限。
2. 团队成员只能编辑自己被授权的数据，默认不可修改项目级计划基准。
3. 只读访客只能查看，不允许新增、编辑、删除、审批。
4. 系统管理员不自动拥有任何项目经理权限；如需介入，应通过单独的负责人转移或系统级操作入口完成。
5. 所有关键操作需记录审计日志。

## 4. 项目状态流转

| 状态 | 说明 |
| --- | --- |
| 规划中 | 项目已创建，处于立项或计划阶段 |
| 进行中 | 项目计划已确认，进入执行状态 |
| 监控中 | 执行控制中的重点监控状态，可选显示 |
| 已结束 | 项目已归档并关闭，不允许业务修改 |

推荐状态流转规则：

1. 新建项目后默认进入 `规划中`。
2. 当核心计划完成并确认后，可流转到 `进行中`。
3. 若启用监控状态，可从 `进行中` 进入 `监控中`，也可由系统自动标记。
4. 验收完成、归档完成后方可流转到 `已结束`。

## 6. 功能模块定稿

## 6.1 认证与用户中心

### 功能说明

1. 用户登录、退出、获取当前用户信息
2. 用户注册或由管理员创建账号
3. 用户基本资料维护
4. 角色权限控制
5. 项目级成员授权

### 业务规则

1. 登录后返回访问令牌。
2. 所有接口按角色和项目权限校验。
3. 停用用户不可登录。

## 6.2 项目初始化管理

### 功能说明

1. 新建项目，录入名称、描述、起止时间、生命周期模型
2. 维护项目章程，包括项目目标、范围概述、审批人、主要干系人
3. 选择项目模板，如瀑布模板、敏捷模板
4. 自动创建演示示例项目
5. 查看、搜索、编辑项目列表

### 业务规则

1. 项目名称在同一组织内建议唯一。
2. 生命周期模型影响默认任务模板和视图。
3. 示例项目首次启动自动创建，可由管理员关闭该能力。

## 6.3 团队与权限管理

### 功能说明

1. 项目成员增删改查
2. 项目角色分配
3. 成员状态管理
4. 成员可查看本人任务、工时、通知

### 业务规则

1. 一个成员可参与多个项目。
2. 项目经理可管理所有成员权限。
3. 团队成员默认不能修改项目基准计划。

## 6.4 范围管理

### 功能说明

1. WBS 树形维护
2. 需求清单维护，区分功能需求和非功能需求
3. 范围基准版本管理
4. 范围变更记录

### 业务规则

1. WBS 支持父子层级。
2. 范围基准一经发布，不直接覆盖旧版本。
3. 范围变更必须关联变更请求。

## 6.5 进度管理

### 功能说明

1. 任务与子任务管理
2. 负责人分配
3. 里程碑管理
4. 任务依赖关系维护，支持 `FS/SS/FF/SF`
5. 任务进度更新
6. 甘特图数据输出
7. 关键路径计算
8. 进度偏差预警

### 业务规则

1. 任务支持计划开始、计划结束、实际开始、实际结束。
2. 进度值范围为 `0-100`。
3. 成员只能更新自己负责的任务，项目经理可更新全部任务。
4. 当实际进度明显落后计划进度时触发预警。

## 6.6 成本管理

### 功能说明

1. 预算编制
2. 成本明细分解
3. 成本基准维护
4. 实际成本录入
5. 工时成本换算
6. EVM 指标计算，包括 `PV/EV/AC/CV/CPI/SPI`

### 业务规则

1. 成本可按项目、阶段、任务维度统计。
2. 成本基准发布后需保留版本。
3. 计划成本与实际成本均需可追溯。

## 6.7 质量与测试管理

### 功能说明

1. 质量标准定义
2. 质量活动计划
3. 质量指标维护
4. 测试计划管理
5. 测试用例管理
6. 缺陷跟踪
7. 测试报告生成

### 业务规则

1. 测试用例和缺陷可关联需求、任务或版本。
2. 缺陷状态至少支持 `新建/处理中/已修复/已关闭`。
3. 测试报告需可统计通过率和遗留缺陷数。

## 6.8 配置与变更管理

### 功能说明

1. 配置项版本记录
2. 变更请求提交
3. 变更审批流程
4. 基线创建与维护
5. 配置状态记录
6. 整体变更控制台

### 业务规则

1. 变更请求必须有标题、影响说明、优先级、提出人。
2. 只有项目经理可审批变更。
3. 已审批变更可触发计划版本更新。

## 6.9 风险管理

### 功能说明

1. 风险登记册
2. 风险等级计算
3. 风险矩阵可视化数据
4. 风险应对计划
5. 风险跟踪与关闭

### 业务规则

1. 风险至少包含概率、影响、等级、应对措施、责任人。
2. 风险状态至少支持 `识别/分析/应对/关闭`。
3. 高等级风险应进入仪表盘和预警列表。

## 6.10 沟通协作管理

### 功能说明

1. 沟通矩阵
2. 会议计划
3. 沟通记录与会议纪要
4. 任务评论
5. 附件上传
6. 站内通知

### 业务规则

1. 评论支持 `@成员`。
2. 会议纪要可关联项目、任务、风险、变更。
3. 重要事件应生成通知，如任务分配、审批结果、风险升级。

## 6.11 采购与合同管理

### 功能说明

1. 供应商管理
2. 采购清单管理
3. 合同信息维护
4. 采购执行跟踪
5. 交付验收记录

### 业务规则

1. 合同需记录类型、签订日期、交付物、付款节点。
2. 采购项可关联预算和实际支出。

## 6.12 工时管理

### 功能说明

1. 日工时或周工时填报
2. 工时与任务关联
3. 工时报表统计
4. 预估工时与实际工时对比

### 业务规则

1. 工时必须关联项目和任务。
2. 成员默认只能录入和查看自己的工时。
3. 项目经理可查看所属项目全部工时。

## 6.13 监控与仪表盘

### 功能说明

1. 项目总览仪表盘
2. 项目数量与状态统计
3. 任务完成率统计
4. 成本执行统计
5. 风险统计
6. 待审批变更统计
7. 质量和缺陷趋势图
8. 报表生成

### 业务规则

1. 仪表盘分首页总览和项目级视图。
2. 统计口径需来自实际业务表，不允许手工维护。

## 6.14 日历与视图管理

### 功能说明

1. 项目日历视图
2. 个人日历视图
3. 甘特图视图
4. 风险矩阵视图

### 业务规则

1. 日历数据来源于任务、里程碑、会议、采购交付等业务对象。

## 6.15 项目收尾管理

### 功能说明

1. 验收清单
2. 成果归档
3. 经验教训库
4. 项目总结报告
5. 项目关闭

### 业务规则

1. 未完成验收和归档时不得直接关闭项目。
2. 项目关闭后默认只读。
3. 项目总结报告需包含计划与实际对比、偏差分析、改进建议。

## 6.16 基础平台能力

### 功能说明

1. 全局搜索与过滤
2. 操作日志
3. Excel/PDF 导出
4. 响应式界面
5. 数据字典和枚举管理

### 业务规则

1. 导出对象至少包括任务列表、风险登记册、成本明细、项目报表。
2. 审计日志覆盖增删改、审批、状态流转、导出等关键操作。

## 7. 首页仪表盘指标定稿

1. 项目总数
2. 进行中项目数
3. 项目总任务数
4. 已完成任务数
5. 项目整体完成率
6. 计划成本
7. 实际成本
8. 未关闭风险数
9. 待审批变更数
10. 本周到期任务数

## 8. 演示数据建议

系统首次启动时自动创建示例项目 `智慧校园App开发`，建议包含以下演示数据：

1. 基础项目资料
2. 项目章程
3. 一套 WBS
4. 8 到 12 个任务
5. 2 个里程碑
6. 3 条风险
7. 一组预算与实际成本
8. 2 条变更请求
9. 3 条工时记录
10. 1 份总结报告草稿

## 9. 交付建议

若希望本次范围一次性确定，建议正式以以下边界作为定版：

1. `V1.0 + V1.1` 进入需求定稿。
2. `V1.2` 不纳入首版验收范围，仅预留扩展接口。
3. 所有模块以“项目”为统一主线，以“角色权限 + 状态流转 + 审计日志”作为共性约束。

以上范围可直接作为产品功能基线。
