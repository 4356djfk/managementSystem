# 软件项目管理系统功能点定稿

## 1. 产品定位

本系统面向课程项目、毕业设计、小中型软件项目和内部协作项目，提供从立项、规划、执行、监控到结项的完整项目管理能力。

当前实现已经覆盖：

1. 项目主数据
2. 计划与进度控制
3. 风险与成本
4. 质量与测试
5. 配置与变更
6. 沟通与采购
7. 工时、报表、归档与审计

## 2. 角色与权限

### 2.1 系统角色

1. `SYS_ADMIN`：用户、模板、审计、系统级配置
2. `USER`：业务侧角色，可进入项目工作台

### 2.2 项目角色

1. `PROJECT_MANAGER`
2. `TEAM_MEMBER`
3. `READ_ONLY`

### 2.3 权限原则

1. 系统管理员不自动拥有所有项目的业务操作权
2. 业务用户通过项目成员关系获得项目数据访问权
3. 写操作统一要求认证，且大部分接口要求 `SYS_ADMIN` 或 `USER`

## 4. 前端现状

前端目前包含两个主要工作区：

1. `dashboard`：项目列表、系统管理、全局能力入口
2. `editor/:projectId`：项目全功能编辑器

其中项目编辑器已经整合了范围、进度、成本、质量、测试、沟通、采购、配置、工时、报表和归档相关能力。

## 5. 部署现状

当前项目已支持容器化一键部署：

1. MySQL
2. Redis
3. Spring Boot 后端
4. Vue 前端

启动命令：

```bash
docker compose up -d --build
```
