# 软件项目管理工具数据库结构设计

## 1. 文档目标

本文档基于已确认的功能点和 API 接口，给出软件项目管理工具的数据库结构设计方案，作为后续数据库建模、实体类编写、DAO 开发和联调的基线文档。

默认数据库建议：

1. 数据库类型：`MySQL 8.x`
2. 字符集：`utf8mb4`
3. 排序规则：`utf8mb4_unicode_ci`
4. 主键类型：`BIGINT`
5. 删除策略：优先使用逻辑删除
6. 时间字段：统一使用 `datetime`

## 2. 设计原则

1. 以 `project_id` 作为大多数业务表的统一主线。
2. 所有核心业务表保留审计字段：`created_by`、`created_at`、`updated_by`、`updated_at`。
3. 所有需要逻辑删除的表保留 `is_deleted` 字段。
4. 状态类字段统一使用英文枚举值，前端显示再做中文映射。
5. V1.0 和 V1.1 表结构一次设计到位，V1.2 只预留扩展字段和关联能力。

## 3. 命名约定

### 3.1 表命名

1. 使用小写下划线风格，如 `project_task`
2. 关联表使用 `主体_关联对象` 命名，如 `project_member`
3. 日志表使用 `_log` 结尾，如 `change_request_log`

### 3.2 通用字段

大多数业务表建议包含以下通用字段：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| created_by | bigint | 创建人 |
| created_at | datetime | 创建时间 |
| updated_by | bigint | 更新人 |
| updated_at | datetime | 更新时间 |
| is_deleted | tinyint(1) | 是否删除，0 否 1 是 |

## 4. 核心 ER 关系概览

核心关系建议如下：

1. 一个 `user` 可参与多个 `project`
2. 一个 `project` 对应一个 `project_charter`
3. 一个 `project` 对应多个 `project_member`
4. 一个 `project` 对应多个 `wbs_node`
5. 一个 `project` 对应多个 `requirement`
6. 一个 `project` 对应多个 `project_task`
7. 一个 `project_task` 可有多个子任务
8. 一个 `project` 对应多个 `risk_register`
9. 一个 `project` 对应多个 `change_request`
10. 一个 `project` 对应多个 `cost_plan` 和 `cost_actual`
11. 一个 `project_task` 对应多个 `timesheet`
12. 一个 `project` 对应多个 `report_record`

## 8. 表关系说明

### 8.1 用户与项目

1. `sys_user` 与 `project_info` 通过 `owner_id` 建立项目负责人关系
2. `sys_user` 与 `project_info` 通过 `project_member` 建立多对多成员关系

### 8.2 项目与任务

1. `project_info` 与 `project_task` 是一对多
2. `project_task` 通过 `parent_task_id` 自关联形成任务树
3. `project_task` 与 `milestone` 是多对一
4. `project_task` 与 `task_dependency` 形成前后置依赖

### 8.3 项目与计划基线

1. `project_info` 与 `scope_baseline` 是一对多
2. `project_info` 与 `baseline_record` 是一对多
3. `change_request` 批准后可关联生成新的基线版本

### 8.4 项目与成本工时

1. `project_task` 与 `timesheet` 是一对多
2. `cost_actual` 可来源于 `timesheet` 或采购数据
3. `cost_plan` 与 `procurement_item` 可建立预算映射

## 10. 索引与性能建议

1. 所有外键字段建立普通索引。
2. 高频列表查询字段建立组合索引，如任务表的 `project_id + status + assignee_id`。
3. 日志表和通知表按时间建立索引，必要时可按月归档。
4. 报表类和快照类字段使用 `longtext` 存储 JSON，避免频繁改表。
5. 对 `project_task`、`operation_log`、`notification` 这类高频表优先优化分页索引。
